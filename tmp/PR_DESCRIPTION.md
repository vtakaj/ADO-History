# fetchの既定プロジェクト化と周辺整備

## PR の概要

fetch コマンドの既定プロジェクト前提化に合わせて設定読み込み・ログ・出力仕様を更新し、テストとドキュメントの整合を取っています。合わせて開発環境/支援ツールの構成ファイルを整理しています。

## 変更点

- Claude関連の設定/コマンド/テンプレートと devcontainer 構成ファイルを削除
- fetch の引数処理を既定プロジェクト前提に変更し、設定検証と利用例を更新
- .env 読み込み方法を簡素化し、404 エラー時のログに環境値を含めるよう変更
- 作業記録テーブルの出力を調整（チケットIDの # 付与、表示対象チケットの抽出と担当者指定の追加）
- テスト実行時の作業ディレクトリ分離や期待値の調整を実施
- 仕様書/ユーザーストーリー/計画ドキュメントを更新
