# PR コードレビュールール

このプロジェクトでは、プルリクエストのコードをレビューし、問題点や改善点を指摘します。

## 目的

プルリクエストのコードをレビューし、以下の観点から問題点や改善点を指摘してください。

## チェック観点

### 1. コードの正確性

- タイポやシンタックスエラーがないか
- 不要なコードや重複した処理が含まれていないか

### 2. ロジックの妥当性

- 条件分岐やループ処理が正しく実装されているか
- ビジネスロジックが意図した動作をしているか

### 3. エラーハンドリング

- 例外やエラーの処理が適切に行われているか
- エラーメッセージに機密情報が含まれていないか

### 4. パフォーマンス

- 無駄な処理が含まれていないか
- パフォーマンスに影響するような重い処理や非効率なアルゴリズムがないか

### 5. コードの可読性

- 命名規則が統一されているか（変数名、関数名など）
- コメントが必要十分に記載されているか

### 6. 再利用性

- 共通処理が関数やモジュールとして適切に切り出されているか
- DRY 原則（Don't Repeat Yourself）に違反していないか

## レビューの結果形式

以下のフォーマットで指摘してください。

**問題点**: 問題の概要（簡潔に記述）

- **該当箇所**: ファイル名・該当コード
- **改善案**: 修正方法の提案

## レビューの流れ

1. プロジェクト全体を俯瞰し、関連するモジュールや影響範囲を把握する
2. ファイル単位でコードを確認し、上記の観点ごとに指摘を行う
3. 必要に応じて、具体的なコード例を挙げて改善案を提示する

## 出力先

- レビュー結果は `tmp/PR_REVIEW.md` に保存する
- `tmp/` フォルダは git 管理から除外されているため、一時ファイルとして安全に使用できる
