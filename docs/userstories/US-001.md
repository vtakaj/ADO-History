# Extract ticket history from Azure DevOps

## Story
As a system
I want to fetch ticket status change history using the Azure DevOps API
So that I can automatically track developer work records

## Acceptance criteria
- [x] Connect to the Azure DevOps API ✅ (US-001-BE-001)
- [x] Fetch the ticket list for a specified project ✅ (US-001-BE-002)
- [x] Fetch status change history for each ticket (timestamp, status before, status after, changed by) ✅ (US-001-BE-003)
- [x] Fetch ticket ID, title, and assignee information ✅ (US-001-BE-002)
- [x] Fetch ticket details (type, priority, estimate, etc.) ✅ (US-001-BE-004)
- [x] Show appropriate error messages on API connection errors ✅ (US-001-BE-001, BE-005 - comprehensive error handling implemented)
- [x] Save fetched data to local JSON files ✅ (US-001-INF-002)
- [ ] Accuracy evaluation has been performed (95%+ accuracy requirement)

## Notes
Use the Azure DevOps REST API to fetch status change history for Work Items. Authentication uses a Personal Access Token (PAT). When fetching large amounts of data, pagination and rate limits must be considered.

## Implemented features (US-001-BE-005)
- **Comprehensive error handling**: detailed guidance for all HTTP errors (4xx, 5xx) and network errors
- **Exponential backoff retry**: efficient automatic retry (gradually increasing up to 300 seconds)
- **Rate limit handling**: set appropriate wait time by parsing the Retry-After header
- **Checkpoint/recovery**: automatic recovery after interruption
- **Timestamped logs**: add timestamps to all process logs
- **Configurable retry parameters**: flexible settings via environment variables
